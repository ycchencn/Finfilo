# 新闻金融影响结构化分析

## 分析目标
对给定新闻进行深度结构化分析，生成标准化JSON输出，用于金融市场影响评估。

## 分析维度及要求

### 1. digest（新闻概要）
- **长度**：50-100字
- **要求**：重新组织语言，客观概括核心信息
- **禁止**：直接复制原文句子
- **重点**：突出时间、主体、事件、结果等关键要素

### 2. tags（关键词标签）
- **数量**：3-5个
- **类型**：行业、政策、技术、事件、人物等核心概念
- **标准**：精准反映新闻主题，便于分类检索

### 3. relation_level（市场关联度）
- **评分范围**：0-10分
- **0分标准**：无明确股市影响或未涉及具体上市公司
- **高分标准**：直接影响A股、港股、美股上市公司的政策、事件或商业动态
- **注意**：仅关注具体公司股价影响，非宏观市场情绪

### 4. bullish_level（利好程度）
- **前提条件**：relation_level > 0时才可赋值
- **评分范围**：-10-10分  
- **0分标准**：对相关股票无明显正面影响
- **-10分标准**：对相关股票有明显利空负面，比如加息/降息预期落空
- **10分标准**：对相关股票有明显利好
- **高分标准**：显著提升公司估值或盈利能力的积极因素
- **关联性**：必须与relations_stocks中的股票直接相关

### 5. relations_stocks（关联股票列表）
**触发条件**：relation_level > 0时必填

### 6. 股票筛选标准：

$prompt_155th

### 输出格式（严格 JSON，禁止任何额外文本）
{
  "digest": "字符串，新闻概要",
  "tags": ['美联储政策', '特朗普', '半导体板块', '美债收益率', '美股'],
  "relation_level": 1,
  "bullish_level": 1,
  "relations_stocks": [
    { "code": "股票代码", "name": "公司名称" },
    { "code": "股票代码", "name": "公司名称" }
  ]
}

---

输出规则：
- 仅输出一个合法 JSON 对象，不加任何解释、注释、前缀或 ```json 分隔符。
- 若 relation_level 为 0，则 relations_stocks 必须为 []。
- 所有字段必须存在，不可省略。

新闻发布时间：
$news_time

用于分析的新闻内容：
$news_content
